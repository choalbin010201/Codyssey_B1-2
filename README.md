# Mission02. AI 브랜드 광고 영상 제작

## 0. 제출 결과 요약

본 과제에서는 **AI 홍보 포스터 생성 서비스 `Poster Coach`**를 주제로 10초 분량의 세로형 브랜드 광고 영상을 제작했다.

- 최종 영상: [`final_video.mp4`](./final_video.mp4)
- 영상 길이: 10초 이내
- 화면 비율: 9:16 세로형
- 사용 도구: ChatGPT 이미지 생성, Kling AI, CapCut
- 포함 요소: AI 생성 시각 요소, 자막, 배경음악
- 직접 촬영 여부: 직접 촬영 없음
- 유료 스톡 사용 여부: 유료 스톡 이미지/영상 사용 없음

---

# 1. 브랜드 아이덴티티 정의

## 1-1. 브랜드명

```text
Poster Coach
```

## 1-2. 서비스 한 줄 설명

```text
소상공인이 가게 정보만 입력하면 AI가 홍보 포스터 문구와 디자인 구성을 빠르게 제안하는 서비스
```

## 1-3. 타겟 사용자

| 항목 | 내용 |
|---|---|
| 주요 타겟 | 소상공인, 자영업자, 작은 카페/식당 운영자 |
| 사용 상황 | SNS, 당근마켓, 네이버 플레이스 등에 올릴 홍보 포스터가 필요할 때 |
| 문제 상황 | 디자인 경험이 부족하고, 홍보 문구 작성이 어렵고, 빠르게 홍보물을 만들어야 함 |

## 1-4. 톤앤매너

| 항목 | 내용 |
|---|---|
| 영상 분위기 | 따뜻함, 현실적, 신뢰감, 문제 해결 중심 |
| 색감 | 베이지, 크림, 브라운, 따뜻한 카페 조명 |
| 스타일 | 현실적인 시네마틱 광고, 부드러운 카메라 움직임 |
| 메시지 톤 | 과장된 광고보다 실용적이고 친근한 도움 제공 |

## 1-5. USP

Poster Coach의 핵심 차별점은 다음과 같다.

```text
디자인 지식이 없어도, 가게 정보만 입력하면 AI가 홍보 포스터를 빠르게 완성해준다.
```

즉, 단순히 이미지를 만드는 서비스가 아니라, 소상공인이 가장 어려워하는 **홍보 문구 작성, 포스터 구성, SNS 게시용 결과물 제작**을 한 번에 도와주는 서비스로 설정했다.

## 1-6. 핵심 메시지

```text
홍보 문구가 어려운 소상공인도 Poster Coach를 사용하면 빠르게 홍보 포스터를 완성할 수 있다.
```

# 5. 프롬프트 수정 전/후 기록

## 5-1. 수정 사례 1: Scene 2 서비스 화면

### 수정 전 프롬프트 요약

```text
카페 사장님이 노트북을 보고 있고, 노트북 화면에 AI 서비스가 포스터를 생성하는 장면
```

### 문제점

- 인물과 배경이 많이 보이면서 노트북 화면의 서비스명이 충분히 강조되지 않았다.
- 브랜드명 `Poster Coach`와 핵심 문구가 작게 보여 서비스 광고로서 전달력이 약했다.
- 10초 영상에서는 시청자가 빠르게 정보를 이해해야 하므로 화면이 더 단순해야 했다.

### 수정 후 프롬프트 요약

```text
노트북 화면을 중심으로 Poster Coach 웹페이지를 크게 보여주는 제품 광고식 클로즈업 장면
```

### 수정 이유

서비스의 핵심 기능과 브랜드명을 명확히 보여주기 위해 인물 중심 장면에서 **웹페이지 중심 장면**으로 변경했다.  
그 결과 `Create posters with AI in seconds`, `Generate Poster`, 템플릿 카드가 더 잘 보이게 되어 광고 메시지 전달력이 높아졌다.

---

## 5-2. 수정 사례 2: Scene 1 인물 일관성

### 수정 전 프롬프트 요약

젊은 남성 카페 사장님이 노트북 앞에서 고민하는 장면

### 문제점

광고의 짧은 길이에서는 인물이 바뀌면 같은 사장님의 문제 해결 과정으로 인식되기 어렵다.

스토리의 일관성을 위해 Scene 1의 인물을 Scene 3과 같은 여성 카페 사장님으로 수정했다.  
이를 통해 “고민하던 사장님 → 서비스 사용 → 만족”이라는 흐름이 더 자연스럽게 연결되었다.

# 8. 크레딧/대기열 제약 고려

AI 영상 생성 도구는 무료 플랜에서 크레딧과 대기 시간이 제한될 수 있다.  
이를 고려하여 다음과 같이 설계했다.

# 11. 핵심 기술 원리 적용

## 11-1. Text-to-Image와 Image-to-Video의 차이

| 기술 | 설명 | 본 프로젝트에서의 사용 |
|---|---|---|
| Text-to-Image | 텍스트 프롬프트를 기반으로 정지 이미지를 생성하는 방식 | 각 씬의 기준 이미지 생성에 사용 |
| Image-to-Video | 이미지를 기반으로 움직임을 추가해 짧은 영상으로 변환하는 방식 | Kling AI에서 장면별 영상 생성에 사용 |
| Text-to-Video | 텍스트만으로 바로 영상을 생성하는 방식 | 이번 프로젝트에서는 사용하지 않음 |

이번 프로젝트에서는 **Text-to-Image → Image-to-Video** 방식을 선택했다.  
그 이유는 10초 광고에서 장면의 구도와 인물, 웹페이지 가독성이 중요했기 때문이다.

Text-to-Video를 바로 사용하면 장면이 예측하기 어렵고, 인물이나 텍스트가 흔들릴 가능성이 높다.  
반면 Text-to-Image로 먼저 기준 이미지를 만들면 구도와 색감, 인물, 웹페이지 화면을 먼저 검수할 수 있고, 이후 Image-to-Video로 필요한 만큼만 움직임을 추가할 수 있다.

## 11-2. 프롬프트 변경으로 인한 결과 변화

| 구분 | 프롬프트 A | 프롬프트 B | 변화 |
|---|---|---|---|
| Scene 2 | 인물과 노트북이 함께 보이는 장면 | 노트북 화면만 크게 보이는 웹페이지 클로즈업 | 서비스명과 핵심 기능이 더 명확해짐 |
| Scene 1 | 남성 사장님 | Scene 3과 같은 여성 사장님 | 스토리 연속성이 좋아짐 |
| Scene 3 | 단순 결과 장면 | 스마트폰 SNS 게시물 + 웃는 사장님 | AI 결과물의 실제 활용 장면이 더 잘 전달됨 |

변화가 발생한 이유는 프롬프트에서 **시각적 우선순위**를 바꿨기 때문이다.  
예를 들어 Scene 2에서는 “사람이 노트북을 보는 장면”보다 “웹페이지 클로즈업”을 강조하자, 결과 이미지에서 서비스 UI가 더 크게 표현되었다.

## 11-3. 캐릭터/스타일 일관성 제어 방법

| 제어 대상 | 적용 방법 |
|---|---|
| 캐릭터 | Scene 3의 여성 사장님 이미지를 기준으로 Scene 1 인물을 수정 |
| 색감 | warm cozy cafe, beige, cream, brown 계열을 반복 사용 |
| 화면비 | 모든 이미지와 영상에서 vertical 9:16 지정 |
| 카메라 움직임 | slow push-in, subtle motion 중심으로 통일 |
| 영상 톤 | 현실적이고 시네마틱한 광고 스타일 반복 지정 |
| 텍스트 안정성 | 웹페이지 장면은 움직임을 최소화하여 글자 왜곡을 줄임 |
| 후처리 | CapCut에서 자막 스타일과 장면 길이를 통일 |

# 15. 심층 인터뷰 답변

## 15-1. 서로 다른 도구 결과물을 통합할 때 생긴 불일치는 어떻게 보정했는가?

이미지 생성 도구와 Kling AI, CapCut을 함께 사용하면서 장면별 색감, 인물 일관성, 화면 비율 차이가 생길 수 있었다.

이를 보정하기 위해 다음 기준을 사용했다.

| 불일치 요소 | 보정 기준 및 절차 |
|---|---|
| 해상도/비율 | 모든 에셋을 9:16으로 생성하고, CapCut에서도 세로형 프로젝트로 통일 |
| 색감 | 따뜻한 카페 조명, 베이지/브라운 계열을 반복 프롬프트에 포함 |
| 인물 | Scene 1과 Scene 3의 사장님을 같은 여성 인물로 맞춤 |
| 텍스트 가독성 | Scene 2는 노트북 화면 중심으로 재생성 |
| 움직임 차이 | Kling에서 과한 움직임을 피하고 slow push-in 중심으로 통일 |
| 자막 | CapCut에서 동일한 폰트 스타일과 위치로 통일 |

가장 중요한 보정 기준은 **광고 메시지가 10초 안에 이해되는가**였다.  
그래서 Scene 2는 웹페이지가 가장 잘 보이도록 재구성했다.

## 15-2. 같은 스토리보드를 다른 도구로 만들면 무엇이 달라질 것으로 예상하는가?

| 도구 방식 | 예상 차이 |
|---|---|
| Kling AI | 이미지 기반 움직임이 자연스럽고 광고 장면 제작에 유리하지만 무료 크레딧 제한이 있음 |
| Pika | 빠르게 생성하기 쉽지만 화면 텍스트나 세부 UI 가독성이 떨어질 수 있음 |
| Runway | 품질과 제어가 좋을 수 있지만 무료 사용 제한이 더 크게 느껴질 수 있음 |
| CapCut만 사용 | 생성형 움직임은 약하지만 이미지 슬라이드와 줌인만으로 안정적인 제출 가능 |

품질 관점에서는 Kling AI가 적합하고, 비용과 안정성 관점에서는 CapCut 이미지 슬라이드 방식이 대체안이 될 수 있다.

## 15-3. 영상 생성 크레딧이 부족해진다면 어떻게 전략을 바꿀 것인가?

크레딧이 부족하다면 영상 생성이 꼭 필요한 장면만 Kling AI로 만들고, 나머지는 CapCut 편집으로 대체한다.

| 상황 | 대체 전략 |
|---|---|
| 크레딧이 1회만 남은 경우 | Scene 3 결과 장면만 영상 생성, Scene 1·2는 이미지 줌인 |
| 크레딧이 0인 경우 | 이미지 3장을 CapCut에서 줌인/전환 효과로 연결 |
| 텍스트 화면이 깨지는 경우 | Scene 2는 영상 생성하지 않고 정지 이미지 + 자막으로 처리 |
| 인물 움직임이 이상한 경우 | 움직임을 줄이고 손/얼굴이 적게 변하는 컷 사용 |

가장 중요한 장면은 Scene 3이다.  
결과물이 SNS에 올라가고 사장님이 만족하는 장면이 서비스 효과를 가장 직접적으로 보여주기 때문이다.

## 15-4. 60초 버전을 15초로 줄여야 한다면 어떻게 재구성할 것인가?

60초 버전을 15초로 줄인다면 설명 장면을 줄이고, 핵심 흐름만 남긴다.

| 유지할 장면 | 이유 |
|---|---|
| 문제 장면 | 시청자가 공감할 수 있는 시작점 |
| 서비스 화면 | Poster Coach의 기능과 USP 전달 |
| 결과 장면 | 사용 후 효과와 만족감 전달 |

삭제하거나 축소할 장면은 다음과 같다.

| 삭제/축소 대상 | 이유 |
|---|---|
| 자세한 기능 설명 | 15초 안에서는 설명보다 시각적 이해가 중요 |
| 템플릿 여러 개 소개 | 핵심 메시지와 직접 관련이 적음 |
| 긴 나레이션 | 짧은 광고에서는 자막이 더 효율적 |

15초 버전 구성은 다음과 같이 설계할 수 있다.

```text
0~4초: 홍보 문구를 고민하는 사장님
4~10초: Poster Coach 웹페이지에서 포스터 생성
10~15초: 완성된 포스터가 SNS에 게시됨
```

이번 과제는 10초 영상이므로 위 구조를 더 압축해 3초, 4초, 3초로 구성했다.

# 17. 결론

본 프로젝트에서는 `Poster Coach`라는 AI 홍보 포스터 생성 서비스를 대상으로 10초 브랜드 광고 영상을 제작했다.

영상은 소상공인이 홍보 포스터 제작에 어려움을 느끼는 문제 상황에서 시작해, Poster Coach 웹페이지를 통해 AI가 포스터를 생성하는 과정을 보여주고, 최종적으로 완성된 포스터가 SNS에 게시되는 결과로 마무리된다.

제작 과정에서는 Text-to-Image로 장면별 기준 이미지를 만들고, Image-to-Video로 움직임을 추가한 뒤, CapCut에서 자막과 배경음악을 통합했다.  
이를 통해 짧은 시간 안에 브랜드의 타겟, 문제 해결 방식, 핵심 메시지를 전달하는 AI 브랜드 광고 영상을 완성했다.
